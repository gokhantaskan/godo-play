# Impeccable Design Context — GodoPlay

## Design Context

### Users

Casual gamers browsing for cross-platform multiplayer games to play with friends. They're not deeply technical — they want to quickly discover games, check platform availability, and understand multiplayer options without friction. The experience should feel effortless, like browsing a curated collection rather than querying a database.

### Brand Personality

**Bold, playful, vibrant.** GodoPlay has energy and personality — it's a discovery tool that should feel fun to use, not just functional. The warm cream background and blue primary already set a friendly, approachable tone. Lean into that warmth while adding moments of delight.

### Aesthetic Direction

- **Visual tone**: Letterboxd/TMDB-inspired content discovery — clean card layouts, editorial feel, rich media, social/browsable energy
- **Anti-references**: No dark/neon gamer aesthetic (RGB rainbow, aggressive typography). No corporate/sterile SaaS look. No cluttered information-dense layouts like legacy game databases
- **Theme**: Light mode primary (warm cream #fcf6f3). Dark mode planned for the future — design decisions should be theme-aware where practical
- **Color system**: Primary blue (#0792f7) anchors the palette. Green/yellow/red for semantic states. Use `color-mix()` in oklab/srgb for dynamic tints — already established in the codebase

### Typography

- **Body**: Figtree (300–700) — friendly, geometric sans-serif
- **Headings**: Lexend (300–700) — optimized for readability, slightly wider
- **Mono**: Roboto Mono
- Headings use `text-wrap: balance`, body text uses `text-wrap: pretty`

### Component Patterns

- Tailwind v4 with `tw:` prefix + SCSS (ITCSS/BEM)
- Headless components via Reka UI and HeadlessUI
- `color-mix()` for hover states (srgb), tinted backgrounds (oklab), overlays (oklch)
- CSS logical properties preferred (`padding-inline`, `margin-block-start`)
- No `@apply` — extract to custom class if >5 utilities

### Emotional Goals

The interface should evoke:

1. **Delight & discovery** — excitement of finding new games, joy in browsing, playful moments
2. **Confidence & clarity** — trust the information, easy to compare, feel in control
3. **Energy & engagement** — dynamic feel that keeps users exploring

### Design Principles

1. **Content is the hero** — Game artwork, screenshots, and metadata should dominate. UI chrome stays minimal and supportive. Cards, images, and media do the heavy lifting.

2. **Warm over cold** — Maintain the warm neutral palette (cream, soft shadows, friendly typography). Even functional elements should feel approachable, never clinical or sterile.

3. **Playful with purpose** — Animations, color accents, and micro-interactions should add personality without slowing users down. Every playful touch should serve discoverability or usability.

4. **Clarity at a glance** — Casual users scan, they don't study. Information hierarchy must be strong — game title, cover art, platform availability, and multiplayer mode should be immediately parseable.

5. **Progressive detail** — Show the essentials upfront, reveal depth on interaction. Cards → modals → full pages. Don't overwhelm on first glance.

### Accessibility

- Standard best practices: good contrast ratios, keyboard navigation, screen reader support
- Respect `prefers-reduced-motion` for animations
- Consider color-blind-friendly semantic color usage (don't rely on color alone for status)
